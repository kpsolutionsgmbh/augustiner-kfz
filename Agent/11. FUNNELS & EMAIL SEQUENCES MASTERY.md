## Teil I — Funnel-Grundlagen

### 1.1 Was ist ein Funnel

> **Ein Funnel ist eine absichtlich gestaltete, strukturierte Abfolge, die einen Interessenten durch eine Reise führt** (Awareness → Interesse → Verlangen → Handlung → Kunde → Botschafter), wobei jede Stufe für maximale Conversion zur nächsten optimiert ist.

#### Komponenten eines vollständigen Funnels

1. **Traffic-Quelle** (Ads, SEO, Social, Partnerschaften — Einstieg oben)
2. **Landing Page** (fängt Aufmerksamkeit, Email-Opt-in ODER Direktverkauf)
3. **Angebots-Sequenz** (Kern → Upsells → Downsells, die Value Ladder hinauf)
4. **Email-Sequenzen** (Nurture, Bildung, Verkauf — automatisierte Beziehung)
5. **Retargeting** (holt Abbrecher zurück, re-engagiert)
6. **Advocacy** (Empfehlungen, Bewertungen, Wiederholungskäufe)

**Brunson:** „Ein Funnel ist KEIN Landing Page. Ein Funnel ist ein SYSTEM: Traffic → Seite 1 → Angebot → Email → Upsell → Repeat. Zusammen konvertiert das System 15–25 % (vs. eine einzelne Seite bei 2–5 %)."

#### Warum Funnels entscheidend sind

- **Einzelne Sales-Email:** 0,5–1,5 % Conversion (kalte Liste)
- **Vollständiger Funnel** (5-Email-Indoctrination → 30-Tage-Nurture → 5-Email-Angebotssequenz): **8–18 % Conversion** (gleiche Liste)
- **12x Unterschied** — Funnel-System vs. isolierte Email

### 1.2 Value Ladder

**Brunson (Dotcom Secrets):** Die Value Ladder ist eine Abfolge von Angeboten mit steigendem Wert (niedrig → mittel → hoch → sehr hoch).

```
TIER 5 — CONTINUITY/BACKEND (97–997 €/Monat Recurring)
  Ziel: Lifetime Value (wiederkehrende Einnahmen)
  Conversion: 20–50 % (jedes Tier → Continuity)

TIER 4 — HIGH-TICKET (5.000–25.000 €)
  Ziel: Premium (tiefe Transformation, Done-With-You)
  Conversion: 10–30 % (Core-Kunden → High-Ticket)

TIER 3 — CORE OFFER (997–2.997 €)
  Ziel: Hauptprodukt (profitabel, löst das komplette Problem)
  Conversion: 15–40 % (Frontend-Käufer → Core)

TIER 2 — FRONTEND (97–497 €)
  Ziel: Erste bezahlte Transaktion (Mikro-Commitment)
  Conversion: 8–25 % (Lead-Magnet-Abonnenten → Käufer)

TIER 1 — LEAD MAGNET (Kostenlos oder Tripwire 7–47 €)
  Ziel: Email erfassen, Wert beweisen
  Conversion: 20–50 % (niedrige Barriere, hohes Volumen)
```

#### Aufstiegs-Beispiel (Kunden-Lebenszyklus)

```
Monat 1:  KOSTENLOS Lead Magnet (Email erfasst)
Monat 2:  Tripwire 47 € (erster Kauf, Vertrauen beginnt)
Monat 3:  Frontend 297 € (Wert bewiesen, Aufstieg)
Monat 6:  Core 1.997 € (starke Beziehung, Hauptangebot)
Monat 12: High-Ticket 10.000 € (tiefe Transformation)
Monat 13+: Continuity 297 €/Monat (laufend, wiederkehrend)

GESAMT LTV: 47 + 297 + 1.997 + 10.000 + (297 × 24 Monate Ø) = 19.472 €
vs. EINZELVERKAUF: 1.997 € (nur Core, keine Leiter)
VALUE LADDER MULTIPLIKATOR: 9,8x LTV
```

**Brunsons Prinzip:** „Nicht sofort Tier 3 verkaufen (schwer bei kaltem Traffic). Tier 1 leicht verkaufen (niedrige Barriere), die Beziehung pflegen (Sequenzen, Wert beweisen) und natürlich aufsteigen lassen."

### 1.3 Customer Value Journey

**Ryan Deiss (DigitalMarketer):** 8 Stufen von Awareness bis Advocacy. Jede braucht anderen Content, andere Angebote, andere Emotionen.

| Stufe | Kundenstatus | Ziel | Funnel-Element | Emotion | Metrik |
|-------|-------------|------|----------------|---------|--------|
| **1 — Awareness** | Kennt dich nicht | Aufmerksamkeit | Ad Creative | Neugier | CTR 1–5 % |
| **2 — Engagement** | Entdeckt, erkundet | Kontakt erfassen | Lead-Gen-Page + Lead Magnet | Interesse | Opt-in 20–50 % |
| **3 — Subscribe** | Email erfasst | Permission festigen | Indoctrination-Sequenz | Dankbarkeit | Open Rate 40–70 % |
| **4 — Convert** | Engagierter Abonnent | Erster Kauf | Tripwire-Angebot | Begeisterung | CVR 8–25 % |
| **5 — Excite** | Neuer Kunde | Erwartungen übertreffen | Produkt-Lieferung + Überraschung | Freude | Completion 80 %+ |
| **6 — Ascend** | Zufriedener Kunde | LTV erhöhen | Upsell-Sequenzen | Verlangen | Ascension 30–60 % |
| **7 — Advocate** | Begeisterter Fan | Amplifikation | Empfehlungsprogramm, Reviews | Stolz | Referral 10–30 % |
| **8 — Promote** | Super-Fan | Partnerschaft | Affiliate-Programm | Unternehmergeist | Affiliate Revenue 10–40 % |

**Fehlende Stufe = ein Leck** im Funnel. Diagnostizieren, welche Stufe schwach ist, und diese spezifisch fixen.

### 1.4 Funnel-Ökonomie

**Belcher:** „Ein profitabler Funnel = einfache Mathematik: LTV > CAC."

#### LTV berechnen (Lifetime Value)

```
LTV = (AOV × Kauffrequenz × Kundenlebensdauer) + Upsells

Beispiel:
  AOV: 1.997 € (Core-Angebot)
  Kauffrequenz: 1,2x/Jahr
  Kundenlebensdauer: 2,3 Jahre
  Upsells Ø: 800 €

  LTV = (1.997 × 1,2 × 2,3) + 800 = 5.511 + 800 = 6.311 €
```

#### CAC berechnen (Customer Acquisition Cost)

```
CAC = Gesamte Marketing-Ausgaben ÷ Neue Kunden

Beispiel:
  Marketing/Monat: 17.000 € (Ads + Tools + Team + Content)
  Kunden/Monat: 34

  CAC = 17.000 ÷ 34 = 500 €/Kunde
```

#### LTV:CAC-Verhältnis

```
LTV: 6.311 € ÷ CAC: 500 € = 12,6:1
```

| Verhältnis | Bewertung | Aktion |
|-----------|-----------|--------|
| < 3:1 | ❌ Unprofitabel | LTV erhöhen (Upsells, Continuity) ODER CAC senken |
| 3:1–5:1 | ⚠️ Profitabel aber dünn | Langsam skalieren |
| 5:1–10:1 | ✅ Gesund | Moderat-aggressiv skalieren |
| > 10:1 | ✅✅ Außergewöhnlich | Massiv skalieren |

#### Payback Period (CAC-Rückgewinnungszeit)

```
Payback = CAC ÷ Monatlicher Umsatz pro Kunde

Beispiel: 500 € ÷ 200 €/Monat = 2,5 Monate
```

| Payback | Bewertung |
|---------|-----------|
| < 3 Monate | ✅✅ Exzellent (Cashflow schnell positiv, aggressiv skalieren) |
| 3–6 Monate | ✅ Gut |
| 6–12 Monate | ⚠️ Langsam (enger Cashflow) |
| > 12 Monate | ❌ Problem (VC-abhängig) |

### 1.5 Funnel vs. Website

| # | Aspekt | Website | Funnel |
|---|--------|---------|--------|
| 1 | Navigation | Mehrere Menüs (10–20 Links) | Nur linear (nur „nächster Schritt") |
| 2 | Ziel | Mehrzweck (informieren, branden, verkaufen) | Ein einziges Ziel (DIESES Angebot konvertieren) |
| 3 | Conversion | 0,5–2 % typisch | 8–25 % (10–50x besser) |
| 4 | Optimierung | Schwierig (was optimieren bei Multi-Purpose?) | Einfach (klarer A/B-Test pro Schritt) |
| 5 | Tracking | Vage (Besucher, Bounces) | Präzise (Stufe-für-Stufe-Drop-off) |
| 6 | Mobile | Oft nicht optimiert | Mobile-first (linear = natürlich) |
| 7 | Speed | Langsam (3–8 Sek.) | Schnell (< 2 Sek., vorgeladen) |
| 8 | Personalisierung | Generisch (gleich für alle) | Dynamisch (verhaltensbasiert) |
| 9 | Upsells | Absent | Integriert (Order Bumps, One-Click-Upsells) |
| 10 | Lifecycle | Endet beim Kauf | Läuft weiter (Sequenzen pflegen, Aufstieg, Retention) |

**Brunson:** „Eine Website ist eine Bibliothek (gut zum Stöbern, SCHLECHT zum Verkaufen). Ein Funnel ist eine Verkaufspräsentation (SCHLECHT zum Stöbern, AUSGEZEICHNET zum Verkaufen)."

### 1.6 E5-Methode

**Todd Brown:** Ein systematisches Funnel-Optimierungs-Framework mit 5 Elementen. Wenn der Funnel schlecht konvertiert: Nicht blind neu bauen — diagnostizieren, WELCHES der 5 Elemente schwach ist.

| E | Element | Frage | Fix |
|---|---------|-------|-----|
| **E1** | Desire verstärken | Ist Engagement hoch? (Opens 30 %+, Watch Time 60 %+) | Traumresultat amplifizieren, LF8, sensorische Details |
| **E2** | Authority erhöhen | Ist Vertrauen aufgebaut? (Ist „Wer bist du?" selten?) | Proof stacken, Credentials, Testimonials |
| **E3** | Einwände eliminieren | Sind die Top 5–10 beantwortet? Starke Garantie? | FAQ, Garantie, Umfrage-basierte Identifikation |
| **E4** | Value etablieren | Ist der ROI klar? (3:1+ gezeigt?) | Stack-Präsentation, Vergleiche, ROI transparent berechnen |
| **E5** | Action ermutigen | Ist Dringlichkeit vorhanden? | Echte Deadline, Knappheit, Verlust-Rahmen |

**Prozess:** Das schwächste E identifizieren → gezielt fixen → neu testen → iterieren bis 15 %+ erreicht.

## Teil II — 8 Funnel-Blueprints

### 2.1 Lead-Generierungs-Funnel

#### Wann einsetzen

- ✅ Liste aufbauen (kalter Traffic, noch nicht kaufbereit)
- ✅ Niedrige Awareness (State 3–5 — brauchen Bildung vor einem Pitch)
- ✅ Langer Verkaufszyklus (B2B, High-Ticket)
- ✅ Autoritätsaufbau (Positionierung als Experte über kostenlosen Content)

#### Struktur (3 lineare Seiten)

**Seite 1 — Ad Creative (Traffic-Quelle):**
- Headline: Neugier ODER Nutzen des Lead Magnets
- CTA: „Kostenlos herunterladen", „Zugang sichern" (niedrige Barriere)
- Ziel: Klick (CTR 1–5 % kalt, 5–15 % warm)

**Seite 2 — Opt-in Landing Page (Email erfassen):**
- Headline: Nutzen des Lead Magnets (15–30 Wörter)
- Bullets: 8–12 (Content anteasern)
- Formular: Nur Email (minimale Reibung — jedes zusätzliche Feld kostet -10–15 % Conversion)
- Social Proof: Zahlen, Bewertungen, Mikro-Testimonials
- Erwartete Conversion: 20–50 %

**Seite 3 — Thank-You-Page (Lieferung + optionaler Tripwire):**
- Bestätigung + sofortiger Download-Link
- Optionaler Tripwire: 47–97 € Angebot (One-Time, massiver Rabatt, verwandt mit dem Lead Magnet)
- Tripwire-Conversion: 8–18 %

#### Funnel-Mathematik

```
1.000 Ad-Impressionen
  → 20 Klicks (2 % CTR)
  → 7 Opt-ins (35 % auf Landing Page)
  → 0,84 Tripwire-Verkäufe (12 % auf Thank-You-Page)

Ergebnis:
  Emails: 7 (15 € CAC jeweils bei 105 € Ad-Spend)
  Umsatz: 39,48 € (0,84 × 47 € Tripwire)
  Netto-CAC: (105 - 39) ÷ 7 = 9,43 €/Email (Tripwire senkt CAC um 37 %)
```

### 2.3 VSL-Funnel

**Ladeira (VTSD):** Die dominante Struktur für Infoprodukte. VSL konvertiert 3,9x besser als Sales Letters (4,7 % vs. 1,2 % bei identischen Angeboten).

#### 4-Schritte-Struktur

**Schritt 1 — Native Ad (Taboola, Facebook — kalter Traffic):**
- Headline: Extreme Neugier (Doc 6 Frage-, Geheimnis-, Entdeckungs-Typen)
- Erwarteter CTR: 1,5–3 %

**Schritt 2 — Advertorial (800–1.200 Wörter — Brücke von Ad → VSL):**
- Kritisch: Nicht direkt von Ad → VSL springen (80 %+ Bounce bei kaltem Traffic)
- Das Advertorial bildet (3–5 Min.), baut Glaubwürdigkeit auf, wärmt kalten Traffic auf
- Ohne Advertorial: VSL-Completion 15–25 %. Mit Advertorial: 45–65 % (2–3x höher)
- Conversion Advertorial → VSL: 30–50 %

**Schritt 3 — VSL-Seite (30–45 Min. Video):**

Layout-Regeln:
- Video zentriert, groß, 80 % der Bildschirmbreite
- Autoplay AN (stumm), Untertitel AN
- KEINE Navigation, KEIN Menü, KEINE Sidebar — null Ablenkungen
- CTA-Button VERSTECKT bis Minute 35–38, DANN sticky

**Ladeiras kritisches Timing (getestet mit 2.400 VSLs):**

| CTA erscheint bei | Conversion |
|-------------------|-----------|
| Minute 0 (sofort sichtbar) | 2,1 % (zu früh, kein Wert geliefert) |
| **Minute 35–38** | **4,7 % (optimal — Wert geliefert, Reziprozität)** |
| Minute 40+ | Nur 40 % Completion (zu spät, viele weg) |

**VSL-Script-Struktur:**
```
Min 00–02: Hook (Pattern Interrupt, Versprechen, Roadmap)
Min 02–08: Diagnose (Problem identifiziert, Selbst-Check)
Min 08–25: Bildung (3 Säulen gelehrt, umsetzbar, großzügig)
Min 25–35: Beweis (Video-Testimonials, Daten, Garantie)
Min 35–43: Angebot (Brunson Stack, Preis, Überraschungs-Bonus, Dringlichkeit)
Min 43–45: Abschluss (Recap, finale Dringlichkeit, Dankbarkeit)
```

**Bildung:Pitch-Verhältnis:** 78 % Bildung / 18 % Pitch / 4 % Abschluss

**Schritt 4 — Checkout-Seite:**
- Minimale Felder (Name, Email, Karte, PLZ — max. 5)
- Garantie wiederholt
- Mehrere Zahlungsmethoden (Kreditkarte mit Raten, PayPal, Sofortüberweisung etc.)

### 2.4 Webinar-Funnel

**Brunson (Expert Secrets, Perfect Webinar):** 90-minütige Live/automatisierte Struktur.

#### 5 Stufen

**Stufe 1 — Registrierungsseite (30–60 % Conversion):**
- Headline: Spezifisches Versprechen (was sie lernen)
- Bullets: 5–8 (die 3 Geheimnisse + Boni anteasern)
- Datum/Uhrzeit: Spezifisch (erzeugt Dringlichkeit)
- Formular: Name + Email

**Stufe 2 — 3 Indoctrination-Emails (vor dem Webinar):**

| Email | Timing | Inhalt |
|-------|--------|--------|
| #1 Bestätigung | Sofort | Kalender-Link, Bullets was gelernt wird, Live-Bonus |
| #2 Wert-Lieferung | 24h vorher | Selbst-Diagnose als Vorbereitung, Live vs. Replay Vorteile |
| #3 Letzte Erinnerung | 1h vorher | Prominenter Link, Tech-Test, Social Proof „340 sind schon drin" |

No-Show-Rate: 40–60 % typisch. Indoctrination-Emails reduzieren No-Shows um 20–30 %.

**Stufe 3 — Live-Webinar (90 Min.):**
- Min 0–10: Origin Story
- Min 10–55: Drei Geheimnisse
- Min 55–75: Stack-Präsentation
- Min 75–90: Q&A
- Live-Conversion: 10–25 %

**Stufe 4 — Replay-Sequenz (3 Emails, 48-Stunden-Fenster):**

| Email | Timing | Inhalt | Conversion |
|-------|--------|--------|-----------|
| #1 | Sofort nach Webinar | Replay verfügbar, 48h Deadline, Live vs. Replay Unterschiede | — |
| #2 | 24h danach | Entscheidungs-Framework, FAQ, Self-Qualification Checklist | — |
| #3 | 2h vor Ablauf | Finale Dringlichkeit, verbleibende Plätze, quantifizierter Verlust | — |

Replay-Conversion: 3–10 % (niedriger als Live, aber Volumen kompensiert)

**Gesamt-Conversion:** 100 Registrierte → 40 live (10–25 % = 4–10 Verkäufe) + 60 Replay (3–10 % = 1,8–6 Verkäufe) = **5,8–16 gesamt**

### 2.5 Product-Launch-Funnel

**Jeff Walker (Product Launch Formula):** Ein Launch ist eine SEQUENZ (10–14 Tage, 15–25 Emails, 3–4 Videos, ein Crescendo der Vorfreude).

#### 4 Phasen, 12–14 Tage

**Phase 1 — Pre-Pre-Launch (Tage 1–3, 3 Emails):**
- Ziel: Bewusstsein schaffen (nicht das Produkt — einen mentalen Samen pflanzen)
- Email #1: Chancen-Enthüllung (89 % stecken fest, warum, die Lücke)
- Email #2: Video #1 (12-Min-Diagnose, Selbst-Check der fehlenden Säulen)
- Email #3: Video #2 (Case Study, Transformation nachvollziehbar)
- Noch NICHTS verkauft — reine Bildung, Vertrauensaufbau

**Phase 2 — Pre-Launch (Tage 4–9, 6–8 Emails):**
- Tiefe Bildung (Videos 3–5, jede Säule im Detail)
- Extreme umsetzbare Bildung, Case Studies, Peak-Engagement
- Wert geliefert: 2.000–3.000 € Consulting-Äquivalent KOSTENLOS
- IMMER NOCH kein Angebot → maximale Vorfreude

**Phase 3 — Launch (Tage 10–12, 6–8 Emails):**
- Warenkorb öffnet, Angebot präsentiert, 72h-Fenster
- Launch-Email #1: Kompletter Stack, 27 Plätze, Sonntag Deadline, Entscheidungs-Framework
- Emails #2–6: Verschiedene Winkel (Case Study, Video-Testimonials, FAQ, 24h-Countdown, 6h-FOMO)
- Kumulative 72h-Conversion: 20–40 %

**Phase 4 — Post-Launch (Tage 13–14, 2 Emails):**
- 12h vor Schluss: Extreme Dringlichkeit
- Nach Schluss: Ausverkauft-Bestätigung, nächste Kohorte Info, Dankbarkeit

**Gesamt PLF:**
- 17–22 Emails, 4–5 Videos, 12–14 Tage
- **Conversion: 20–40 %** (die HÖCHSTE Methode)
- vs. einzelne Email: 1–3 % → **10–15x Unterschied**

## Teil III — 5 Email-Sequenz-Typen

### 3.1 Soap Opera Sequenz

**Chaperon (Tiny Little Businesses):** Story-basierte Emails mit extremem Engagement. Eine Story-Arc über 5–7 Emails mit Cliffhangern. 40–70 % Open Rates vs. 15–25 % für Promo-Emails.

#### Struktur (5–7 Emails)

| Email | Funktion | Inhalt | Cliffhanger |
|-------|----------|--------|-------------|
| **#1 — Bühne setzen** | Protagonist + Problem vorstellen | Name, Situation (Avatar-Match), Leiden, erschöpfende Versuche | „Bis Januar 2023. Maria fand… [nicht enthüllt]" |
| **#2 — Steigende Handlung** | Lösungsversuch, erstes Hindernis | Was sie fand (teilweise enthüllt), Skepsis, Entscheidung zu versuchen | „Tag 1: Gleich. Tag 3: Sie wachte auf… [etwas passierte]" |
| **#3 — Annäherung Höhepunkt** | Durchbruch, Hoffnung | Tag 3–7 Progression, vorsichtige Hoffnung | „Monat 1: Der Schmerz war [dramatisches Ergebnis, nicht genau enthüllt]" |
| **#4 — Höhepunkt** | Transformation komplett | Monat 1–3 vollständig, Familie/Leben wiederhergestellt | „Marias Entscheidung zu teilen… [in #5]" |
| **#5 — Auflösung + Lektion** | Story abgeschlossen, Lektion, Angebot | Reise-Recap, „du kannst auch", SUBTILES Angebot am Ende | — |
| **#6–7 (optional)** | Direktes Angebot | Falls nicht gekauft: transparenter Übergang zum Verkaufen | — |

**Open Rates im Verlauf:** Email #1: 35–55 % → #2: 45–65 % → #3: 50–70 % (PEAK) → #4–5: 45–60 %

### 7.6 Kontinuierlicher Optimierungs-Loop

Optimierung ist kein einmaliger Vorgang — es ist kontinuierliches Testen, Iterieren und ständiges Verbessern.

#### Der Loop

```
1. BASELINE messen (aktueller Stand: 28 % Opens, 6 % CTR, 9 % Conv.)
2. HYPOTHESE formulieren („Werden Neugier-Betreffzeilen Opens boosten?")
3. TESTEN (A/B — Version A Kontrolle, Version B Neugier-Betreff, je 500)
4. MESSEN (A: 28 % Opens, B: 34 % → B gewinnt mit +6 Punkten)
5. IMPLEMENTIEREN (B wird neue Kontrolle)
6. WIEDERHOLEN (nächste Hypothese: „CTA-Button grün vs. rot?")

→ Ein ENDLOSER LOOP: Min. 1 Test/Woche, 50–60 kumulative
  Verbesserungen/Jahr
```

#### Test-Prioritäten (nach Impact)

| Priorität | Was testen | Impact |
|-----------|-----------|--------|
| **1 — Betreffzeilen** | 10+ Variationen (Neugier vs. Nutzen vs. Frage vs. Dringlichkeit) | +20–80 % mehr Opens |
